¿Qué son clases en React?

En React, los componentes son las unidades fundamentales para construir interfaces de usuario. En versiones anteriores de React, se utilizaban principalmente clases para definir componentes. Estos componentes de clase extendían la clase React.Component y permitían gestionar el estado y el ciclo de vida de un componente. Sin embargo, con la evolución de React, se introdujeron los "componentes funcionales" y los "hooks". Los componentes funcionales son funciones de JavaScript que pueden aceptar propiedades y devolver elementos React. Los hooks, como useState y useEffect, permiten a los componentes funcionales gestionar el estado y realizar efectos secundarios, lo que antes solo era posible con las clases.

Principales eventos de React

onClick:

Este evento se activa cuando se hace clic en un elemento. Es comúnmente utilizado para manejar acciones de usuario como hacer clic en un botón.

onChange:


Se activa cuando el valor de un elemento de formulario, como un input o un select, cambia. Este evento es valioso para rastrear y responder a los cambios en los datos de entrada del usuario.

onSubmit:


Este evento se dispara cuando se envía un formulario. Es útil para realizar acciones como la validación de formularios y el envío de datos.

onMouseOver y onMouseOut:


onMouseOver se activa cuando el puntero del ratón se coloca sobre un elemento. onMouseOut se activa cuando el puntero del ratón sale de un elemento. Estos eventos son útiles para efectos visuales o información adicional al pasar el ratón sobre un elemento.

onKeyPress:

Se activa cuando una tecla se presiona mientras un elemento tiene el foco. Puede ser utilizado para capturar y manejar eventos de teclado.

onFocus y onBlur:

onFocus se activa cuando un elemento obtiene el foco. onBlur se activa cuando un elemento pierde el foco. Estos eventos son útiles para realizar acciones cuando un usuario interactúa con o deja un elemento. Estos eventos permiten a los desarrolladores controlar la interactividad en una aplicación React y responder a las acciones del usuario sin tener que proporcionar código específico.
